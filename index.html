연구 동기와 목적

사건보다 해석이 감정과 행동을 더 크게 만듭니다

같은 시험 결과도 해석 방식에 따라 반응이 달라집니다. 이 프로젝트는 그 차이를 기록하고, 학생이 왜곡된 사고를 스스로 발견하게 만드는 데 초점을 둡니다.

왜 이 프로젝트를 만들었는가

CBT는 사건보다 생각이 감정과 행동을 만든다고 봅니다. 하지만 혼자 기록하면 어디가 사실이고 어디서부터 왜곡인지 판단하기 어렵습니다.

그래서 학생이 기록만 해도 AI가 왜곡 패턴, 되돌아볼 질문, 대안 사고를 제안하는 도구를 구현했습니다.

핵심 연구 질문

1
학생이 왜곡된 사고를 스스로 찾을 수 있는가

전문 상담사 없이도 사고 패턴을 점검할 수 있게 합니다.

2
생각-감정-행동 연결을 데이터로 볼 수 있는가

기록이 쌓일수록 자기이해와 메타인지가 강화되게 합니다.

3
생성형 AI가 대안 사고까지 제안할 수 있는가

Gemini API로 왜곡 분류와 질문 생성을 자동화합니다.

목표 1. 왜곡된 생각 발견

대표 왜곡을 학생이 직접 확인합니다.

목표 2. 연결 고리 시각화

상황-생각-감정-행동을 한눈에 추적합니다.

목표 3. 메타인지 강화

대안 사고 훈련으로 자기 점검을 강화합니다.

프로젝트 계획

랜딩 설명과 실제 동작을 한 화면에 묶었습니다

기록 입력, AI 분석, 시각화, 주간 리포트까지 하나의 브라우저 앱 흐름으로 연결했습니다.

주간 AI 리포트

최근 7일 기록을 자동 요약합니다.

1 1단계: 입력 + API 연동

입력 후 즉시 AI 반응을 구현합니다.

2 2단계: 저장 + 시각화

저장·차트·히트맵을 연결합니다.

3 3단계: 주간 리포트 + 완성도

리포트·디자인·오류를 마무리합니다.

7가지 인지 왜곡 패턴

Gemini 분석에서도 아래 분류 체계를 기준으로 응답하도록 제한했습니다.

Gemini API Setup

Gemini 연결 시 실제 모델 분석으로 전환됩니다

브라우저에서 Gemini REST API를 직접 호출합니다. 키가 없거나 호출에 실패하면 로컬 분석으로 자동 전환됩니다.

보안 주의

키는 localStorage에 저장되므로 개인 테스트용 구성이며, 운영 시에는 서버 프록시가 필요합니다.

실행 방식

`file://` 대신 로컬 서버에서 열면 API 호출 오류를 줄일 수 있습니다.

폴백 로직

Gemini 실패 시에도 기록, 그래프, 리포트는 로컬 엔진으로 계속 동작합니다.

연결 설정

기본 모델은 `gemini-2.5-flash`로 설정되어 있습니다.

저장된 API 키 없음
Gemini를 연결하면 기록 분석 결과에 모델 기반 설명과 질문이 들어갑니다.
실제 동작 영역

학습 불안을 기록하고 생각 패턴을 바로 분석합니다

폼 입력과 대화형 기록을 모두 지원하며, Gemini 실패 시에는 즉시 로컬 분석으로 이어집니다.
